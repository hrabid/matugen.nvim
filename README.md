# matugen.nvim

> One wallpaper. Consistent colors across Neovim and VS Code — powered by [matugen](https://github.com/InioX/matugen)

`matugen.nvim` is a Material You colorscheme for Neovim built around a dedicated template system. It maps your matugen-generated colors directly to Neovim highlight groups through a semantic palette — no color recomputation, no compromises for other editors

If you already use [matugen-editors](https://github.com/Senal-D-A-Gunaratna/matugen-editors) for VS Code, you get cross-editor consistency for free with the same source colors driving both

---

## Requirements

- Neovim 0.10+
- [matugen](https://github.com/InioX/matugen) installed and configured

---

## Setup

**1. Copy the template**

Copy [nvim-template.jsonc](nvim-template.jsonc) to your matugen templates folder

**2. Add to your matugen `config.toml`**

```toml
[templates.neovim]
input_path = "~/.config/matugen/templates/nvim-template.jsonc"
output_path = "~/.config/matugen/themes/nvim-template.jsonc"
post_hook = "pkill -SIGUSR1 nvim"
```

**3. Install the plugin via `lazy.nvim`**

```lua
return {
  {
    "Senal-D-A-Gunaratna/matugen.nvim",
    lazy = false,
    priority = 1000,
    opts = {
      -- Path to the JSONC file generated by matugen
      jsonc_path = vim.fn.expand("~/.config/matugen/themes/nvim-template.jsonc"),
    },
    config = function(_, opts)
      require("matugen").setup(opts)
      vim.cmd.colorscheme("matugen")
    end,
  },
}
```

## VS Code Parity

If you use my [matugen-editors](https://github.com/Senal-D-A-Gunaratna/matugen-editors) setup, you can point the plugin at your existing `code-colors.jsonc` instead of the nvim-template — as long as it shares the same structure. both editors will simultaneously theme from from the same colorpalate.

---

## Live Reload

The plugin listens for `SIGUSR1`. The matugen `post_hook` above handles this automatically, but you can also trigger it manually:

```bash
pkill -SIGUSR1 nvim
```

Or reload in neovim without the signal:

```vim
:MatugenReload
```

---

## Customization

The plugin is built to be extended.

Colors are mapped through a centralized semantic palette in `lua/matugen/palette.lua`. When you add support for a new plugin or UI component, everything stays consistent because all highlights derive from the same source

To add your own highlight groups or plugin support, see [Creating Custom Templates](doc/TEMPLATES.md)
